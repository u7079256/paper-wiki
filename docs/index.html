Core Concept

LLM Wiki, not RAG

RAG retrieves text chunks at query time and stitches an answer on the fly — quality depends on chunking and recall. paper-wiki takes the opposite approach: Claude reads every source page-by-page upfront (including appendices), writes structured notes with citations, then synthesizes cross-source concepts and research gaps. The compiled wiki is the product — it works across sessions, and opens as a knowledge graph in Obsidian.

核心理念

LLM Wiki，不是 RAG

RAG 每次提问都临时检索文本片段再拼答案，质量取决于切分和召回；paper-wiki 反过来 — Claude 事先把每篇源文献从头读到尾（包括附录），写成结构化笔记并标注出处，再跨源综合出概念条目和研究空白。编译好的 wiki 本身就是产出，跨会话可用，用 Obsidian 打开就是一张知识图谱。

Installation

One command — it's a plugin

Claude Code fetches and manages the plugin. No manual copying into ~/.claude/, no permission prompts.

安装

两条命令，装完即用

Claude Code 自动拉取并管理 plugin，不用手动往 ~/.claude/ 里复制任何东西。

Claude Code Claude Code 
> /plugin marketplace add u7079256/paper-wiki
> /plugin install paper-wiki@paper-wiki

Update later with /plugin marketplace update paper-wiki. Manage all plugins via /plugin.

SSH host-key error? The plugin system clones via SSH. If you see No ED25519 host key is known for github.com, run once:
ssh-keyscan -t ed25519 github.com >> ~/.ssh/known_hosts

Don't want a plugin? You can git clone this repo and run the bootstrap below — bootstrapped projects are self-contained.

后续更新：/plugin marketplace update paper-wiki。

遇到 SSH host-key 报错？ plugin 系统通过 SSH 克隆仓库。如果报 No ED25519 host key is known for github.com，执行一次：

不想装 plugin 也行 — 直接 git clone 本仓库，用下面的 bootstrap 脚本创建项目。创建出来的项目自带 /wiki-* 命令，不依赖 plugin。

Quick Start

Bootstrap a new wiki project

The bootstrap script creates a full project skeleton: raw/ + wiki/, commands, agents, OCR scripts, and variant-specific templates.

快速开始

创建 wiki 项目

bootstrap 脚本会生成完整的项目骨架：raw/ + wiki/ 两层结构、命令、agent、OCR 脚本，以及根据变体渲染好的模板。

Windows PowerShell 
PS> .\scripts\bootstrap_new_wiki.ps1 -NewPath D:\my-wiki -Topic my-topic `
     -ProjectName "My Wiki" -Variant research      # or course
macOS / Linux 
$ bash scripts/bootstrap_new_wiki.sh --path ~/my-wiki --topic my-topic \
     --name "My Wiki" --variant research      # or course

Then start Claude Code in that folder and run /wiki-init.

No GPU needed to try it out. See examples/QUICKSTART.md for a 5-minute walkthrough using the no-OCR WebFetch path (Claude reads the HTML directly from arXiv).

然后在该目录启动 Claude Code，运行 /wiki-init 初始化。

不需要 GPU 也能体验。见 examples/QUICKSTART.md，5 分钟跑完全流程 — Claude 直接通过 WebFetch 读 arXiv 论文的 HTML，不用配 OCR。

Commands

Slash command reference

Inside a bootstrapped wiki project, these commands are available as /wiki-*. With the plugin installed, use /paper-wiki:wiki-* from anywhere.

命令

命令速查

在 bootstrap 创建的项目里，这些命令以 /wiki-* 形式调用。装了 plugin 后，在任何目录用 /paper-wiki:wiki-*。

CommandWhat it does
/wiki-initOne-time setup: fill topic + seeds (research) / unpack + inventory (course)
/wiki-compileRead new raw/ material, write paper/lecture notes, synthesize concepts
/wiki-ask <q>Read-only query: answers only from the compiled wiki, cites sources, says "not in wiki" when absent
/wiki-search-latestFind recent papers to import research
/wiki-critique <file>Adversarial review: holes, overclaims, wrong formulas
/wiki-verify-noveltyCheck a claimed gap against prior work research
命令功能
/wiki-init初始化（仅首次）：填写主题和种子论文（research）/ 解包并清点材料（course）
/wiki-compile读取 raw/ 中的新材料，编译笔记，综合概念或主题
/wiki-ask <问题>只读查询：仅从已编译的 wiki 中回答，标注出处，没有就说没有
/wiki-search-latest搜索最新相关论文 research
/wiki-critique <文件>对抗性审查：找漏洞、过度声明、公式错误
/wiki-verify-novelty验证声称的研究空白是否真的没人做过 research
